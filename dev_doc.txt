大闸蟹精准养殖智能决策咨询系统 — 软件开发文档
1. 软件概述
本系统是一款面向大闸蟹养殖户及技术人员的垂直领域智能咨询平台。系统利用**检索增强生成（RAG）**技术，通过匹配本地专业知识库来增强大语言模型（LLM）的回答能力，解决养殖过程中信息不对称、决策不科学的问题 。
2. 系统架构设计
系统采用前后端分离架构，核心逻辑遵循 RAG 流程。
2.1 技术栈选型
前端：Vue 3 + Element Plus + Echarts（用于行情可视化）。
后端：Python 3.9 + FastAPI（高性能异步框架）。
向量数据库：ChromaDB（用于存储和检索分块后的养殖知识向量）。
AI 模型接口：DeepSeek-V3 API / GPT-4o-mini（提供自然语言理解与生成能力）。
中间件：LangChain（编排 RAG 工作流）。
3. 核心模块详细设计
3.1 知识库构建模块（数据预处理）
数据采集：整合《大闸蟹养殖技术规范》、病害诊断手册及相关学术论文 。
文本清洗：去除 PDF/网页中的无关噪声（如页码、广告），保留核心养殖知识 。
文本分块（Chunking）：
策略：RecursiveCharacterTextSplitter（递归字符分割）。
块大小：500 字符；重叠：50 字符。确保病害描述与治疗方案在同一语义块内。
向量化（Embedding）：调用 text-embedding-3-small 或 BGE-M3 模型将文本转为向量并存入 ChromaDB 。
3.2 智能检索与对话模块
用户查询向量化：将用户输入（如“大闸蟹第二次蜕壳需要补充什么？”）转化为向量。
语义检索：在 ChromaDB 中计算余弦相似度，检索出最相关的 Top-3 知识片段。
Prompt 增强：
System Prompt: "你是一位拥有20年经验的大闸蟹养殖专家。请仅根据提供的上下文回答问题，若库中无相关内容，请告知用户咨询人工专家，不得编造。"
结果输出：LLM 结合背景知识生成精准回复。
3.3 功能模块设计
问答中心：支持文本/语音输入，展示包含参考来源的 AI 回复。
生长周期看板：根据当前月份，自动提示大闸蟹所处的阶段（如“扣蟹期”或“成熟期”）及管理要点 。
行情监测：通过 API 获取并展示各规格大闸蟹（如 3 两公、2 两母）的市场价格走势 。

5. 关键业务流程
用户提问：前端发送请求至后端 /api/chat。
检索：后端在 ChromaDB 检索匹配养殖手册内容。
调用 LLM：将检索内容作为 context，拼接用户问题，发送至 LLM API 。
响应：后端接收流式回复（Streaming），实时推送到前端。
6. 系统测试计划
准确性测试：准备 50 条常见养殖问题，对比“原始模型”与“RAG系统”的回答准确率。
延迟测试：测试从提问到首字弹出的时间，目标控制在 2 秒以内。
压力测试：模拟 10 人同时在线咨询，确保 FastAPI 服务稳定。
